# S1.1 macOS Input Spike QA

Date: 2026-06-12

Scope: macOS capture, injection, self-injection filtering, permissions, and screen topology.

## Automated Verification

- `cargo check --example mac_input_spike`
- `cargo clippy --example mac_input_spike -- -D warnings`
- `cargo fmt --check`
- `cargo test`
- `cargo clippy --all-targets -- -D warnings`
- `npm run build`

## Manual Commands

Run from `src-tauri`:

```bash
cargo run --example mac_input_spike
cargo run --example mac_input_spike -- --topology
cargo run --example mac_input_spike -- --inject-move
cargo run --example mac_input_spike -- --inject-click
cargo run --example mac_input_spike -- --verify-self-filter
```

## Permission Notes

- When running the spike from iTerm2 or Terminal, grant Input Monitoring and Accessibility to that terminal app.
- When running the Tauri app, grant permissions to the FlowLink app identity.
- Returning focus to FlowLink refreshes the permission state.

## Manual QA Checklist

- [ ] Capture prints move events.
- [ ] Capture prints left/right/middle down and up events.
- [ ] Capture prints wheel events with expected vertical/horizontal direction.
- [ ] `--inject-move` moves the cursor and prints median/p95 call latency.
- [ ] `--inject-click` clicks the current cursor target after the 2 second countdown.
- [ ] `--verify-self-filter` reports zero captured injected events.
- [ ] `--topology` prints all displays with bounds, scale factor, and primary flag.

## Known Limitations

- Back/Forward side-button injection is best-effort on macOS because CoreGraphics exposes only left/right/center button constants; FlowLink still sets the button number field for other buttons.
- Capture shutdown now requests the CFRunLoop to stop and joins briefly, but full lifecycle management will be revisited when SessionController owns capture.
